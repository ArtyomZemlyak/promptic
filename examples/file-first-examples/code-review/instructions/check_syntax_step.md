# Check Syntax and Style

Verify code follows language-specific syntax rules and project style guidelines.

## Syntax Checks

- No syntax errors or warnings
- Proper indentation and formatting
- Consistent naming conventions
- Appropriate use of language features

## Style Guidelines

- Follow project's linter configuration
- Match existing code patterns
- Use meaningful variable and function names
- Keep functions focused and concise

## Tools

- Run linter: `pylint` or `ruff`
- Check formatting: `black` or `autopep8`
- Verify type hints if project uses them
