# Suggest Fix

Provide concrete suggestions for resolving identified issues.

## Fix Suggestions Should Include

1. **Specific solution**: Exact code changes or approach
2. **Rationale**: Why this fix addresses the issue
3. **Alternatives**: Other valid approaches if applicable
4. **Testing**: How to verify the fix works

## Suggestion Format

- Use code blocks to show before/after examples
- Reference relevant documentation or standards
- Link to similar fixes in the codebase when possible
- Explain trade-offs if multiple solutions exist

## Best Practices

- Prefer simple, readable solutions
- Maintain consistency with existing codebase
- Consider long-term maintainability
- Ensure suggestions are feasible to implement
