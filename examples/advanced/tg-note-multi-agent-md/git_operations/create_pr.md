# Создание Pull Request

## Задача

Создать Pull Request для изменений в knowledge base.

## Процесс создания PR

1. **Подготовка к PR**
   - Убедиться что все изменения закоммичены
   - Проверить что изменения запушены в ветку
   - Убедиться что ветка актуальна с main/master

2. **Создание PR**
   - Создать Pull Request через GitHub API или интерфейс
   - Написать понятное название
   - Добавить описание изменений
   - Указать связанные issues если есть

3. **Описание PR**
   - Краткое описание изменений
   - Список измененных файлов
   - Объяснение причин изменений
   - Примеры если уместно

4. **Проверка PR**
   - Убедиться что PR создан
   - Проверить что описание корректно
   - Убедиться что все изменения включены

## Формат описания PR

```markdown
## Описание
Краткое описание изменений

## Изменения
- Изменение 1
- Изменение 2

## Связанные issues
#123

## Чеклист
- [ ] Код проверен
- [ ] Тесты пройдены
- [ ] Документация обновлена
```
