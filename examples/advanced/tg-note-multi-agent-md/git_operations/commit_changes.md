# Коммит изменений

## Задача

Создать Git коммит с изменениями в knowledge base.

## Процесс коммита

1. **Проверка изменений**
   - Проверить статус репозитория (`git status`)
   - Определить какие файлы изменены
   - Убедиться что изменения корректны

2. **Подготовка коммита**
   - Добавить файлы в staging (`git add`)
   - Проверить что добавлено правильно
   - Убедиться что нет лишних файлов

3. **Создание коммита**
   - Написать понятное сообщение коммита
   - Указать тип изменения (feat, fix, docs, etc.)
   - Добавить описание изменений
   - Создать коммит (`git commit`)

4. **Проверка коммита**
   - Проверить что коммит создан
   - Убедиться что сообщение корректно
   - Проверить что все нужные файлы включены

## Формат сообщения коммита

Используй следующий формат:

```
тип: краткое описание

Подробное описание изменений если нужно.
```

Типы:
- `feat`: новая функциональность
- `fix`: исправление
- `docs`: документация
- `style`: форматирование
- `refactor`: рефакторинг
