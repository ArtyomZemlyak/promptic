# Git Workflow

## Общие правила работы с Git

### Коммиты

1. **Сообщения коммитов**: Пиши понятные сообщения на русском или английском
2. **Частота коммитов**: Делай коммиты логически завершенными изменениями
3. **Размер коммитов**: Один коммит = одна логическая задача

### Ветки

- Используй осмысленные названия веток
- Создавай ветки для новых фич или исправлений
- Синхронизируйся с main/master регулярно

### Pull Requests

- Создавай PR для всех изменений в main
- Добавляй описание изменений
- Указывай связанные issues если есть

### Безопасность

- Не коммить чувствительные данные (пароли, ключи)
- Проверяй изменения перед коммитом
- Используй `.gitignore` для исключения ненужных файлов
