# Categorize Issue

Classify each issue into appropriate categories for better tracking and prioritization.

## Issue Categories

- **Bug**: Functionality that doesn't work as expected
- **Security**: Potential vulnerabilities or security concerns
- **Performance**: Code that may cause slowdowns or resource issues
- **Style**: Formatting, naming, or code organization issues
- **Documentation**: Missing or unclear documentation
- **Test**: Missing tests or test quality issues
- **Architecture**: Design or structural concerns

## Severity Levels

- **Critical**: Blocks functionality or introduces security risks
- **High**: Significant impact on functionality or maintainability
- **Medium**: Moderate impact, should be addressed before merge
- **Low**: Minor issues, can be addressed in follow-up PRs
