# Code Review Principles

These principles guide all code review activities.

## Core Values

1. **Be constructive**: Focus on improving code quality, not criticizing the author
2. **Be specific**: Point to exact lines and explain why changes are needed
3. **Be respectful**: Maintain a professional and supportive tone
4. **Be thorough**: Check for security, performance, and maintainability issues

## Review Checklist

- [ ] Code follows project style guidelines
- [ ] Tests are included and passing
- [ ] Documentation is updated
- [ ] No security vulnerabilities introduced
- [ ] Performance considerations addressed
- [ ] Edge cases handled appropriately
