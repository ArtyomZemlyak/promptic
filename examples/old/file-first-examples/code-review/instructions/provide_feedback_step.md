# Provide Structured Feedback

Generate clear, actionable feedback for each identified issue.

## Feedback Structure

For each issue, provide:

1. **Location**: File path and line numbers
2. **Issue type**: Bug, style, performance, security, etc.
3. **Severity**: Critical, high, medium, or low
4. **Description**: Clear explanation of the problem
5. **Suggestion**: Specific recommendation for improvement

## Feedback Guidelines

- Be specific and actionable
- Prioritize critical and high-severity issues
- Group related issues together
- Provide code examples when helpful
- Acknowledge good practices when you see them
